Pipeline History

Per-day breakdown of how each captured frame was decided. Every capture flows through pixel-diff first (cheap check for an empty bassinet, no model needed); non-empty frames go to BIRDEYE, the on-device MobileNetV3 cascade (presence → face → eye-state); only when BIRDEYE bails (no_face_detected / low_confidence / hard error) does the frame fall back to the Cloud API (GPT-4o vision). Cost is cloud calls × $0.01. The model version column shows which BIRDEYE checkpoint(s) produced the BIRDEYE-decided frames that day; percentages are of BIRDEYE's slice (so they sum to ~100%), making it easy to read "version X did most of BIRDEYE's work today". On pre-flip days these labels can be anachronistic because backfill_birdeye_primary.py may have re-tagged historical rows.
